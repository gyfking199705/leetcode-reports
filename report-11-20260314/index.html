🎯 LeetCode 精讲 #11 - 盛最多水的容器

📋 题目描述

来源: LeetCode 第11题
难度: Medium
链接: https://leetcode.cn/problems/container-with-most-water/

题目原文

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

说明：你不能倾斜容器。

示例

示例 1:

输入: height = [1,8,6,2,5,4,8,3,7]
输出: 49
解释: 第1条线(高度8)和第8条线(高度7)之间的容器，宽度为7，高度为min(8,7)=7
      面积为 7 × 7 = 49

示例 2:

输入: height = [1,1]
输出: 1

示例 3:

输入: height = [4,3,2,1,4]
输出: 16
解释: 第0条线和第4条线，宽度4，高度min(4,4)=4，面积16

约束条件

  • n == height.length
  • 2 <= n <= 10^5
  • 0 <= height[i] <= 10^4

💡 解题思路

方法一：暴力枚举（不推荐）

思路描述：

  • 枚举所有可能的左右边界组合
  • 计算每对边界形成的容器面积
  • 取最大值

面积计算公式：

面积 = min(height[left], height[right]) × (right - left)

时间复杂度：O(n²) —— 会超时！

方法二：双指针法 ⭐ 推荐

核心思想：

容器的容量由较短的板子决定。让两个指针从两端向中间移动，每次移动较短的那一端，才有可能找到更大的面积。

为什么移动较短端？

假设当前左指针在位置 L，右指针在位置 R：

  • 当前面积 = min(height[L], height[R]) × (R - L)
  • 如果 height[L] < height[R]（左端较短）

移动右指针的情况：

  • 新宽度 (R' - L) 一定变小（因为R左移了）
  • 新高度 min(height[L], height[R']) ≤ height[L]（因为左端已经是短板）
  • 所以新面积一定小于等于原面积！

移动左指针的情况：

  • 虽然宽度也变小，但高度有可能变大
  • 如果 height[L'] > height[L]，面积就有可能变大！

结论：只有移动较短端，才有可能获得更大的面积。

算法步骤：

  1. 初始化左指针 left = 0，右指针 right = n - 1
  2. 计算当前面积，更新最大值
  3. 比较 height[left] 和 height[right]：
    • 如果 height[left] < height[right]：left++
    • 否则：right--
  4. 重复步骤2-3，直到 left >= right

📝 参考代码

class Solution:
    def maxArea(self, height: List[int]) -> int:
        """
        双指针法求解盛最多水的容器问题
        
        思路：
        - 使用左右双指针从两端向中间移动
        - 每次移动较短的那一端，才有可能找到更大的面积
        - 因为面积受限于较短的边，移动短边才可能找到更高的边
        
        Args:
            height: 整数数组，表示每条线的高度
            
        Returns:
            能容纳的最大水量（最大面积）
        """
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 计算当前容器的宽度
            width = right - left
            
            # 容器的高度由较短的板子决定
            current_height = min(height[left], height[right])
            
            # 计算当前面积
            current_area = width * current_height
            
            # 更新最大面积
            max_area = max(max_area, current_area)
            
            # 移动较短的那一端的指针
            # 为什么？因为面积受限于较短的边，只有移动短边才可能找到更高的边
            if height[left] < height[right]:
                # 左端较短，向左移动左指针
                left += 1
            else:
                # 右端较短（或相等），向右移动右指针
                right -= 1
        
        return max_area


# 更简洁的写法
class Solution:
    def maxArea(self, height: List[int]) -> int:
        left, right = 0, len(height) - 1
        max_area = 0
        
        while left < right:
            # 一行计算面积并更新最大值
            max_area = max(max_area, 
                          (right - left) * min(height[left], height[right]))
            
            # 移动较短端的指针
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area

⏱️ 复杂度分析

方法 时间复杂度 空间复杂度 说明
暴力枚举 O(n²) O(1) 会超时，不推荐
双指针 ⭐ O(n) O(1) 最优解法

双指针分析：

  • 时间复杂度 O(n)：左右指针最多各移动 n 次，总共最多 2n 次操作
  • 空间复杂度 O(1)：只使用了常数个变量

🔑 关键要点

  1. 核心洞察：

    • 容器面积 = min(左高, 右高) × 宽度
    • 宽度在缩小过程中是单调递减的

  2. 移动策略：

    • 移动较长端：新面积一定 ≤ 原面积（宽度↓，高度≤原高度）
    • 移动较短端：才有可能找到更大的高度，从而可能得到更大面积

  3. 为什么正确？

    • 双指针不会错过最优解
    • 对于任意最优解 (i, j)，当两个指针分别位于 i 和 j 时一定会被考虑到

  4. 易错点：

    • 忘记用 min() 取两边高度的较小值
    • 混淆左右指针的移动条件

  5. 面试常问：

    • "为什么移动较短的那一端？" —— 这是考察理解深度的核心问题
