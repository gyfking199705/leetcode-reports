23. 合并K个升序链表

Merge k Sorted Lists — 链表 · 分治 · 堆

📅 2026-04-27 🏷️ 链表 / 堆 / 分治 ⏱️ 预计 25 分钟掌握
📋 题目描述

给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中并返回。

示例
// 输入
lists = [
  [1->4->5],
  [1->3->4],
  [2->6]
]

// 输出
[1->1->2->3->4->4->5->6]
💡 核心观察

这题是 21. 合并两个有序链表 的升级版。k=2 时就是两链表合并，k 很大时需要更高效的策略。关键是如何快速找到当前所有链表头节点中的最小值。

⚖️ 三种解法对比
方法时间空间核心思想
逐一合并 O(k²n) O(1) 用合并两个链表的函数，逐个合并 k 个链表
分治合并 O(kn·log k) O(log k) 两两配对，递归合并，类似归并排序
优先队列 ⭐ O(kn·log k) O(k) 小根堆维护 k 个链表头，每次取最小

最直观的方法：维护一个结果链表，依次与每个输入链表合并。

result ← merge → list[0]
result ← merge → list[1]
result ← merge → list[2]
... 共 k 次合并
⚠️ 为什么慢？

第一次合并：合并 n + n = 2n 个节点
第二次合并：合并 2n + n = 3n 个节点
...
第 i 次合并：合并 i·n + n 个节点

总节点访问次数 = 2n + 3n + ... + kn = n·(k+2)(k-1)/2 ≈ O(k²n)

逐一合并 · Python 
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        
        # 复用合并两个链表的函数
        def mergeTwo(l1, l2):
            dummy = ListNode(0)
            tail = dummy
            while l1 and l2:
                if l1.val <= l2.val:
                    tail.next = l1
                    l1 = l1.next
                else:
                    tail.next = l2
                    l2 = l2.next
                tail = tail.next
            tail.next = l1 if l1 else l2
            return dummy.next
        
        # 逐个合并
        result = lists[0]
        for i in range(1, len(lists)):
            result = mergeTwo(result, lists[i])
        return result

类似归并排序的分治策略：将 k 个链表两两配对合并，递归处理直到只剩一个链表。

💡 为什么快？

每次合并的链表长度相近，避免了"一长一短"的低效情况。

层数 = log₂k，每层访问总节点数 = O(kn)
总复杂度 = O(kn · log k)

分治合并 · Python 
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        if not lists:
            return None
        return self.mergeRange(lists, 0, len(lists) - 1)
    
    def mergeRange(self, lists, left, right):
        if left == right:
            return lists[left]
        if left + 1 == right:
            return self.mergeTwo(lists[left], lists[right])
        
        mid = (left + right) // 2
        l1 = self.mergeRange(lists, left, mid)
        l2 = self.mergeRange(lists, mid + 1, right)
        return self.mergeTwo(l1, l2)
    
    def mergeTwo(self, l1, l2):
        dummy = ListNode(0)
        tail = dummy
        while l1 and l2:
            if l1.val <= l2.val:
                tail.next, l1 = l1, l1.next
            else:
                tail.next, l2 = l2, l2.next
            tail = tail.next
        tail.next = l1 if l1 else l2
        return dummy.next

最优面试解法。维护一个大小为 k 的小根堆，存储每个链表的当前头节点。每次从堆顶取出最小节点接到结果链表，并将该节点的下一个节点入堆。

Min-Heap (size=k)

┌─────┐
│ 1 ←─┼── list[0]: [1→4→5]
│ 2 │    list[1]: [2→6]
│ 3 │    list[2]: [3→7]
└─────┘

取出 1 → result.next = 1
将 4 入堆

┌─────┐
│ 2 ←─┼── ...
│ 3 │
│ 4 │
└─────┘
🎯 堆操作复杂度

每次取最小 + 插入新节点 = O(log k)
总共处理 kn 个节点
总复杂度 = O(kn · log k)

Python 用 heapq，但需要自定义比较（因为直接比较 ListNode 会报错）。技巧：存入 (node.val, idx, node) 三元组。

优先队列 · Python（最优解） 
import heapq

class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        dummy = ListNode(0)
        tail = dummy
        
        # 最小堆: (值, 索引, 节点)
        # 索引用于打破平局，避免比较 ListNode
        min_heap = []
        for i, node in enumerate(lists):
            if node:
                heapq.heappush(min_heap, (node.val, i, node))
        
        while min_heap:
            val, idx, node = heapq.heappop(min_heap)
            tail.next = node
            tail = tail.next
            if node.next:
                heapq.heappush(min_heap, (node.next.val, idx, node.next))
        
        return dummy.next
📝 关键点

1. dummy 节点：简化头节点处理，最后返回 dummy.next
2. 三元组入堆：heapq 比较元素时若值相同会接着比较下一个元素，ListNode 不可比较所以放索引 i 作为 tie-breaker
3. 空链表过滤：入堆前判断 if node，避免空指针
4. 空间：堆大小不超过 k，空间复杂度 O(k)

🎤 面试要点
  1. 先问边界：k=0 怎么办？有空链表怎么办？答：返回 None
  2. 先给暴力：逐一合并 O(k²n)，展示基础能力
  3. 再优化：提到归并思路或堆，这才是面试官想听的
  4. 讲清复杂度：分治和堆都是 O(kn·log k)，分治空间 O(log k) 更优，堆实现更短
  5. 手写 mergeTwo：dummy + tail 双指针，烂熟于心
  6. Python 细节：heapq 不能直接存 ListNode，要包装成 (val, idx, node)
📊 复杂度速查
最优时间复杂度
O(kn·log k)
最优空间复杂度
O(log k)

注：n 为单个链表平均长度，k 为链表数量。分治法空间为递归栈 O(log k)，堆方法空间为 O(k)。
