🎯 LeetCode 精讲 #300 - 最长递增子序列

📋 题目描述

原题链接: https://leetcode.cn/problems/longest-increasing-subsequence/

难度: 中等
标签: 动态规划、二分查找

题目原文

给你一个整数数组 nums，找到其中最长严格递增子序列的长度。

子序列是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

示例

示例 1：

输入：nums = [10,9,2,5,3,7,101,18]
输出：4
解释：最长递增子序列是 [2,3,7,101]，因此长度为 4。

示例 2：

输入：nums = [0,1,0,3,2,3]
输出：4

示例 3：

输入：nums = [7,7,7,7,7,7,7]
输出：1
解释：严格递增要求后一个数必须大于前一个数，所以相同元素不构成递增。

约束条件

  • 1 <= nums.length <= 2500
  • -10^4 <= nums[i] <= 10^4

💡 解题思路

🔍 核心问题分析

这道题的核心难点在于：子序列不要求连续，这意味着我们不能简单地用滑动窗口解决。

关键概念：

  • 子数组：连续的一段
  • 子序列：保持相对顺序，不要求连续（可跳跃选取）

例如：[10,9,2,5,3,7,101,18] 中

  • 子数组 [2,5,3] 是连续的
  • 子序列 [2,3,7,101] 是从数组中按顺序选取的（跳过5）

方法一：动态规划 (O(n²))

状态定义

设 dp[i] 表示：以第 i 个元素结尾的最长递增子序列的长度

⚠️ 注意：这里的状态定义非常关键——"以第i个元素结尾"

状态转移方程

对于每个位置 i，遍历所有前面的位置 j (0 <= j < i):
    如果 nums[j] < nums[i]:
        dp[i] = max(dp[i], dp[j] + 1)
        // 表示可以在以 j 结尾的递增子序列后接上 nums[i]

初始状态：每个元素自身构成长度为1的子序列

dp[i] = 1 (对所有 i)

可视化理解

以 nums = [10,9,2,5,3,7,101,18] 为例：

下标 i nums[i] 前面比它小的元素 dp[i] 转移过程
0 10 无 1 初始值
1 9 无（9<10） 1 初始值
2 2 无（2<9<10） 1 初始值
3 5 2 2 dp[2]+1 = 2
4 3 2 2 dp[2]+1 = 2
5 7 2,5,3 3 max(dp[2],dp[3],dp[4])+1 = 3
6 101 2,5,3,7 4 max(dp[2],dp[3],dp[4],dp[5])+1 = 4
7 18 2,5,3,7 4 max(dp[2],dp[3],dp[4],dp[5])+1 = 4

最终答案是 max(dp) = 4

代码实现

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        n = len(nums)
        if n <= 1:
            return n
        
        # dp[i] 表示以第 i 个元素结尾的最长递增子序列长度
        dp = [1] * n
        
        for i in range(1, n):
            # 遍历所有前面的位置
            for j in range(i):
                if nums[j] < nums[i]:
                    # 如果 nums[j] < nums[i]，可以接在后面
                    dp[i] = max(dp[i], dp[j] + 1)
        
        # 答案是以任意位置结尾的最大值
        return max(dp)

复杂度分析

  • 时间复杂度: O(n²) — 双重循环
  • 空间复杂度: O(n) — dp数组

方法二：二分查找优化 (O(n log n))

核心洞察

考虑维护一个数组 tails，其中 tails[i] 表示：长度为 i+1 的递增子序列的最小末尾元素

为什么要记录"最小末尾元素"？

因为末尾元素越小，后面越容易接上更多元素，这样更容易形成更长的递增子序列。

关键性质

tails 数组是严格递增的！

证明：如果存在 tails[i] >= tails[j] 且 i < j，那长度为 i+1 的子序列末尾元素比长度为 j+1 的还大，这显然矛盾——因为我们总可以用长度为 i+1 的更小末尾构造更优解。

由于 tails 递增，我们可以用二分查找来维护它！

算法步骤

  1. 初始化 tails = []
  2. 对于每个 num 在 nums 中：
    • 如果 num > tails[-1]（比所有末尾都大）：直接追加，子序列长度+1
    • 否则：在 tails 中找到第一个 >= num 的位置，替换它

示例演示

nums = [10,9,2,5,3,7,101,18]

步骤 num tails 数组 操作
1 10 [10] 初始
2 9 [9] 替换10（9<10，更有利于后续扩展）
3 2 [2] 替换9
4 5 [2,5] 5>2，追加
5 3 [2,3] 替换5（找到第一个>=3的位置）
6 7 [2,3,7] 7>3，追加
7 101 [2,3,7,101] 101>7，追加
8 18 [2,3,7,18] 替换101（18<101）

最终 tails 长度为 4，即为答案。

注意：tails 不一定是真正的LIS，但它的长度就是LIS的长度！

代码实现

from bisect import bisect_left

class Solution:
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []  # tails[i] 表示长度为 i+1 的递增子序列的最小末尾元素
        
        for num in nums:
            # 找到 tails 中第一个 >= num 的位置
            pos = bisect_left(tails, num)
            
            if pos == len(tails):
                # num 比 tails 中所有元素都大，可以延长子序列
                tails.append(num)
            else:
                # 替换，保持该长度的最小末尾
                tails[pos] = num
        
        return len(tails)

复杂度分析

  • 时间复杂度: O(n log n) — 每个元素二分查找
  • 空间复杂度: O(n) — tails数组

🔧 完整代码（含测试用例）

from typing import List
from bisect import bisect_left

class Solution:
    """
    LeetCode 300: 最长递增子序列
    
    两种解法：
    1. 动态规划：O(n²) 时间，直观易懂
    2. 二分优化：O(n log n) 时间，面试推荐
    """
    
    # 方法一：动态规划 O(n²)
    def lengthOfLIS_DP(self, nums: List[int]) -> int:
        n = len(nums)
        if n <= 1:
            return n
        
        # dp[i] = 以第 i 个元素结尾的最长递增子序列长度
        dp = [1] * n
        
        for i in range(1, n):
            for j in range(i):
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        
        return max(dp)
    
    # 方法二：二分查找优化 O(n log n)
    def lengthOfLIS(self, nums: List[int]) -> int:
        tails = []
        
        for num in nums:
            pos = bisect_left(tails, num)
            
            if pos == len(tails):
                tails.append(num)
            else:
                tails[pos] = num
        
        return len(tails)


# ========== 测试用例 ==========
if __name__ == "__main__":

    nums1 = [10, 9, 2, 5, 3, 7, 101, 18]
    print(f"示例1: {nums1}")
    print(f"  DP解法: {sol.lengthOfLIS_DP(nums1)}")   # 4
    print(f"  二分解法: {sol.lengthOfLIS(nums1)}")    # 4
    
    # 测试用例2
    nums2 = [0, 1, 0, 3, 2, 3]
    print(f"\n示例2: {nums2}")
    print(f"  DP解法: {sol.lengthOfLIS_DP(nums2)}")   # 4
    print(f"  二分解法: {sol.lengthOfLIS(nums2)}")    # 4
    
    # 测试用例3
    nums3 = [7, 7, 7, 7, 7, 7, 7]
    print(f"\n示例3: {nums3}")
    print(f"  DP解法: {sol.lengthOfLIS_DP(nums3)}")   # 1
    print(f"  二分解法: {sol.lengthOfLIS(nums3)}")    # 1
    
    # 边界测试
    print(f"\n边界测试:")
    print(f"  空数组: {sol.lengthOfLIS([])}")          # 0

💡 面试高频考点

  1. 为什么选择 tails[i] 表示"最小末尾"？

    • 越小的末尾越容易后续扩展，这是贪心思想

  2. tails 数组一定是有序的吗？

    • 是的，严格递增，可以用反证法证明

  3. tails 数组是LIS本身吗？

    • 不一定！tails只保证长度正确，但不保证序列连续

  4. 如何输出LIS的具体序列？

    • 需要额外记录每个元素的前驱，在DP解法中通过回溯得到
