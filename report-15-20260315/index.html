🎯 LeetCode 精讲 #15 - 三数之和

📋 题目描述

原题链接: https://leetcode.cn/problems/3sum/

难度: 中等
标签: 数组、双指针、排序

题目原文

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。

请你返回所有和为 0 且不重复的三元组。

注意： 答案中不可以包含重复的三元组。

示例

示例 1：

输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0
不同的三元组是 [-1,0,1] 和 [-1,-1,2]
注意，输出的顺序和三元组的顺序并不重要

示例 2：

输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0

示例 3：

输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0

约束条件

  • 3 <= nums.length <= 3000
  • -10^5 <= nums[i] <= 10^5

💡 解题思路

🔍 核心洞察：降维 + 去重

这道题是「两数之和」的升级版，从找两个数变成找三个数。

关键思路：

  1. 固定一个数，问题转化为「在剩余数组中找两个数，和为 -固定数」
  2. 排序后使用双指针，将 O(n²) 的两数查找优化到 O(n)
  3. 巧妙去重，跳过重复元素避免结果重复

算法步骤详解

输入: [-1, 0, 1, 2, -1, -4]

第1步: 排序
       [-4, -1, -1, 0, 1, 2]
        ↑                ↑
       left            right

第2步: 固定第一个数，用双指针找剩余两数

固定 -4 (i=0): 需要在 [-1, -1, 0, 1, 2] 中找和为 4 的两数
    双指针: left=-1, right=2 → sum = -1+2 = 1 < 4, left右移
    双指针: left=-1, right=2 → sum = -1+2 = 1 < 4, left右移  
    双指针: left=0, right=2 → sum = 0+2 = 2 < 4, left右移
    left >= right, 结束，无结果

固定 -1 (i=1): 需要在 [-1, 0, 1, 2] 中找和为 1 的两数
    双指针: left=-1, right=2 → sum = -1+2 = 1 == 1 ✓ 找到 [-1, -1, 2]
    跳过重复的 left: [-1, 0, 1, 2]
    跳过重复的 right: [-1, 0, 1, 2]
    继续: left=0, right=1 → sum = 0+1 = 1 == 1 ✓ 找到 [-1, 0, 1]

固定 -1 (i=2): 与前一个相同，跳过（去重）

固定 0 (i=3): 需要在 [1, 2] 中找和为 0 的两数，无解

固定 1 (i=4): 剩余元素不足，结束

📝 代码实现

Python 解法

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        if n < 3:
            return []
        
        # 1. 排序 - 这是使用双指针的前提
        nums.sort()
        result = []
        
        # 2. 固定第一个数
        for i in range(n - 2):  # 留两个位置给双指针
            
            # 剪枝优化1: 当前数已大于0，后续都大于0，三数之和不可能为0
            if nums[i] > 0:
                break
            
            # 去重: 跳过重复的固定数
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            
            # 3. 双指针找剩余两数
            left, right = i + 1, n - 1
            target = -nums[i]  # 需要找到的两数之和
            
            while left < right:
                current_sum = nums[left] + nums[right]
                
                if current_sum == target:
                    # 找到一组解
                    result.append([nums[i], nums[left], nums[right]])
                    
                    # 去重: 跳过重复的左指针元素
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    # 去重: 跳过重复的右指针元素  
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    # 移动指针继续查找
                    left += 1
                    right -= 1
                    
                elif current_sum < target:
                    # 和太小，左指针右移（增大）
                    left += 1
                else:
                    # 和太大，右指针左移（减小）
                    right -= 1
        
        return result

Java 解法

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        int n = nums.length;
        if (n < 3) return result;
        
        // 1. 排序
        Arrays.sort(nums);
        
        // 2. 固定第一个数
        for (int i = 0; i < n - 2; i++) {
            // 剪枝
            if (nums[i] > 0) break;
            
            // 去重
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            // 3. 双指针
            int left = i + 1, right = n - 1;
            int target = -nums[i];
            
            while (left < right) {
                int sum = nums[left] + nums[right];
                
                if (sum == target) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    
                    // 去重
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    
                    left++;
                    right--;
                } else if (sum < target) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        
        return result;
    }
}

C++ 解法

class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> result;
        int n = nums.size();
        if (n < 3) return result;
        
        sort(nums.begin(), nums.end());
        
        for (int i = 0; i < n - 2; i++) {
            if (nums[i] > 0) break;
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            
            int left = i + 1, right = n - 1;
            int target = -nums[i];
            
            while (left < right) {
                int sum = nums[left] + nums[right];
                
                if (sum == target) {
                    result.push_back({nums[i], nums[left], nums[right]});
                    
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;
                    
                    left++;
                    right--;
                } else if (sum < target) {
                    left++;
                } else {
                    right--;
                }
            }
        }
        
        return result;
    }
};

⏱️ 复杂度分析

步骤 时间复杂度 空间复杂度 说明
排序 O(n log n) O(1) / O(log n) 取决于排序算法
外层循环 O(n) O(1) 固定第一个数
双指针 O(n) O(1) 内层双指针遍历
总计 O(n²) O(1) 不含结果存储空间

时间复杂度详细推导：

  • 排序：O(n log n)
  • 外层循环 n 次，内层双指针平均 O(n) 次
  • 总复杂度：O(n log n + n²) = O(n²)

空间复杂度：

  • 排序：原地排序 O(1)（或 O(log n) 递归栈）
  • 结果存储：O(k)，k 为答案数量（不计入算法复杂度）

🔑 关键要点

1. 为什么先排序？

  • 排序是双指针使用的前提
  • 排序后可以根据和的大小关系，决定移动哪个指针
  • 排序让重复元素相邻，便于去重

2. 三重去重机制

┌─────────────────────────────────────────────────────┐
│  第一层：跳过重复的固定数 nums[i]                    │
│  if i > 0 and nums[i] == nums[i-1]: continue       │
│                                                     │
│  第二层：跳过重复的左指针元素 nums[left]             │
│  while left < right and nums[left] == nums[left+1] │
│                                                     │
│  第三层：跳过重复的右指针元素 nums[right]            │
│  while left < right and nums[right] == nums[right-1]│
└─────────────────────────────────────────────────────┘

3. 剪枝优化

# 如果当前固定数已大于0，后续都大于0，不可能找到和为0的三元组
if nums[i] > 0:
    break  # 直接结束循环

4. 双指针移动规则

nums[left] + nums[right] vs target

       小于目标          等于目标           大于目标
           │               │                │
           ▼               ▼                ▼
    ┌────────────┐   ┌──────────┐    ┌────────────┐
    │  left++    │   │ 记录答案  │    │  right--   │
    │ (需要更大)  │   │ 双指针收缩 │    │ (需要更小)  │
    └────────────┘   └──────────┘    └────────────┘

🌟 常见错误

❌ 错误1：去重位置不对

# 错误：在循环开始就去重
if nums[i] == nums[i + 1]:  # 这样写会跳过有效解
    continue

❌ 错误2：忘记移动指针

# 错误：找到答案后不移动指针
if current_sum == target:
    result.append([...])
    # 忘记 left++ 和 right--，导致死循环

❌ 错误3：边界判断错误

# 错误：循环范围
for i in range(n):  # 应该留两个位置给双指针
for i in range(n - 2):  # 正确

🎯 面试高频考点

  1. 时间复杂度证明：为什么是 O(n²)？能否优化？

    • 答案：最优解就是 O(n²)，需要遍历所有可能的第一个数

  2. 空间复杂度：能否做到 O(1)？

    • 答案：可以，排序+双指针就是 O(1) 额外空间

  3. 去重机制：如何确保不重复？

    • 答案：三处去重 + 排序让重复元素相邻

  4. 变体问题：如果数组未排序？如果要求返回下标？

    • 答案：返回下标需要用哈希表，时间 O(n²)，空间 O(n)

🎓 总结

LeetCode 15 是双指针算法的经典应用，核心技巧：

  1. 降维思想：三数之和 → 固定一数 + 两数之和
  2. 排序预处理：为双指针创造使用条件
  3. 三处去重：确保结果不重复
  4. 剪枝优化：提前终止不可能的分支
