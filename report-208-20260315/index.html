LeetCode 208. 实现 Trie (前缀树)

📋 题目描述

Trie（发音类似 "try"），也称为前缀树或字典树，是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补全和拼写检查。

请你实现 Trie 类：

  • Trie() 初始化前缀树对象。
  • void insert(String word) 向前缀树中插入字符串 word。
  • boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false。
  • boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix，返回 true；否则，返回 false。

示例

输入
["Trie", "insert", "search", "search", "startsWith", "insert", "search"]
[[], ["apple"], ["apple"], ["app"], ["app"], ["app"], ["app"]]
输出
[null, null, true, false, true, null, true]

解释
Trie trie = new Trie();
trie.insert("apple");
trie.search("apple");   // 返回 True
trie.search("app");     // 返回 False
trie.startsWith("app"); // 返回 True
trie.insert("app");
trie.search("app");     // 返回 True

提示

  • 1 <= word.length, prefix.length <= 2000
  • word 和 prefix 仅由小写英文字母组成
  • insert、search 和 startsWith 调用次数总计不超过 3 * 10⁴ 次

💡 解题思路

核心概念：什么是 Trie？

Trie 是一种多叉树结构，每个节点代表一个字符，从根节点到某一节点的路径表示一个字符串（前缀）。

关键特性：

  1. 根节点为空：不存储字符
  2. 边代表字符：从父节点到子节点的边标记一个字符
  3. 节点标记终止：用一个布尔值标记是否有单词在此节点结束
  4. 共享前缀：具有公共前缀的单词共享路径

结构示意图

插入 "apple" 和 "app" 后的 Trie：

        (root)
          |
          a
          |
          p
          |
          p  ← isEnd = true (单词 "app" 结束)
          |
          l
          |
          e  ← isEnd = true (单词 "apple" 结束)

算法步骤

1. 节点定义

class TrieNode:
    def __init__(self):
        # 子节点字典：字符 -> TrieNode
        self.children = {}
        # 标记是否有单词在此节点结束
        self.is_end = False

2. 插入操作 (insert)

时间复杂度：O(m)，其中 m 为单词长度

从根节点开始
对于单词中的每个字符 c：
    如果当前节点的 children 中不存在 c：
        创建新节点
    移动到子节点
标记最后一个节点 is_end = True

3. 搜索操作 (search)

时间复杂度：O(m)

从根节点开始
对于单词中的每个字符 c：
    如果当前节点的 children 中不存在 c：
        返回 False
    移动到子节点
返回当前节点的 is_end 值

4. 前缀匹配 (startsWith)

时间复杂度：O(m)

从根节点开始
对于前缀中的每个字符 c：
    如果当前节点的 children 中不存在 c：
        返回 False
    移动到子节点
返回 True（只要能走完前缀就存在）

Python 实现

class TrieNode:
    """Trie 节点定义"""
    def __init__(self):
        # 使用字典存储子节点，键为字符，值为 TrieNode
        self.children = {}
        # 标记是否为某个单词的结尾
        self.is_end = False


class Trie:
    """前缀树实现"""
    
    def __init__(self):
        """初始化 Trie，创建根节点"""
        self.root = TrieNode()

    def insert(self, word: str) -> None:
        """
        插入单词到 Trie
        时间复杂度: O(m)，m 为单词长度
        """
        node = self.root
        for char in word:
            # 如果字符不存在，创建新节点
            if char not in node.children:
                node.children[char] = TrieNode()
            # 移动到子节点
            node = node.children[char]
        # 标记单词结束
        node.is_end = True

    def search(self, word: str) -> bool:
        """
        搜索完整单词
        时间复杂度: O(m)
        """
        node = self._search_prefix(word)
        # 必须存在该节点，且该节点是某个单词的结尾
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        """
        检查是否存在以 prefix 为前缀的单词
        时间复杂度: O(m)
        """
        return self._search_prefix(prefix) is not None
    
    def _search_prefix(self, prefix: str) -> TrieNode:
        """
        辅助方法：搜索前缀，返回最后一个节点或 None
        """
        node = self.root
        for char in prefix:
            if char not in node.children:
                return None
            node = node.children[char]
        return node

优化版本：使用数组替代字典

对于仅包含小写英文字母的情况，可用固定大小数组优化：

class TrieNode:
    def __init__(self):
        # 26个小写字母，初始为 None
        self.children = [None] * 26
        self.is_end = False


class Trie:
    def __init__(self):
        self.root = TrieNode()

    def _char_to_idx(self, char: str) -> int:
        """字符转索引：'a' -> 0, 'b' -> 1, ..."""
        return ord(char) - ord('a')

    def insert(self, word: str) -> None:
        node = self.root
        for char in word:
            idx = self._char_to_idx(char)
            if node.children[idx] is None:
                node.children[idx] = TrieNode()
            node = node.children[idx]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self._search_node(word)
        return node is not None and node.is_end

    def startsWith(self, prefix: str) -> bool:
        return self._search_node(prefix) is not None
    
    def _search_node(self, word: str):
        node = self.root
        for char in word:
            idx = self._char_to_idx(char)
            if node.children[idx] is None:
                return None
            node = node.children[idx]
        return node

Java 实现

class TrieNode {
    // 子节点数组
    TrieNode[] children;
    // 是否为单词结尾
    boolean isEnd;
    
    public TrieNode() {
        children = new TrieNode[26]; // 假设只有小写字母
        isEnd = false;
    }
}

class Trie {
    private TrieNode root;
    
    public Trie() {
        root = new TrieNode();
    }
    
    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) {
                node.children[idx] = new TrieNode();
            }
            node = node.children[idx];
        }
        node.isEnd = true;
    }
    
    public boolean search(String word) {
        TrieNode node = searchPrefix(word);
        return node != null && node.isEnd;
    }
    
    public boolean startsWith(String prefix) {
        return searchPrefix(prefix) != null;
    }
    
    private TrieNode searchPrefix(String prefix) {
        TrieNode node = root;
        for (char c : prefix.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) {
                return null;
            }
            node = node.children[idx];
        }
        return node;
    }
}

class TrieNode {
public:
    TrieNode* children[26];
    bool isEnd;
    
    TrieNode() {
        for (int i = 0; i < 26; i++) {
            children[i] = nullptr;
        }
        isEnd = false;
    }
};

class Trie {
private:
    TrieNode* root;
    
    TrieNode* searchPrefix(string prefix) {
        TrieNode* node = root;
        for (char c : prefix) {
            int idx = c - 'a';
            if (node->children[idx] == nullptr) {
                return nullptr;
            }
            node = node->children[idx];
        }
        return node;
    }
    
public:
    Trie() {
        root = new TrieNode();
    }

        TrieNode* node = root;

            int idx = c - 'a';
            if (node->children[idx] == nullptr) {

            node = node->children[idx];
        }
        node->isEnd = true;
    }
    
    bool search(string word) {
        TrieNode* node = searchPrefix(word);
        return node != nullptr && node->isEnd;
    }

};

📊 复杂度分析

时间复杂度

操作 时间复杂度 说明
insert O(m) m 为单词长度，需遍历每个字符
search O(m) 最坏遍历整个单词
startsWith O(m) 只需匹配前缀

空间复杂度

  • 最坏情况：O(n × m)，n 是单词数，m 是单词平均长度
  • 最优情况：O(总字符数)，共享前缀可大幅减少空间

空间优化对比：

  • 存储 ["apple", "app", "application", "apply"] 共需约 13 个节点
  • 单独存储需 4 + 3 + 11 + 5 = 23 个字符

🎯 关键要点总结

1. Trie 的核心优势

场景 优势
前缀匹配 O(m) 时间，远快于哈希表的 O(n×m)
自动补全 找到前缀后，DFS/BFS 遍历所有子节点
字典序遍历 天然支持按字母顺序输出
拼写检查 快速判断单词是否存在

2. Trie vs 哈希表

特性 Trie 哈希表
精确匹配 O(m) O(1) 平均
前缀匹配 O(m) O(n×m)
空间效率 共享前缀，较优 独立存储
字典序 天然支持 需额外排序

3. 常见变种

  • 压缩 Trie：合并只有一个子节点的链
  • 后缀 Trie：存储所有后缀（用于字符串匹配）
  • 后缀树：压缩后的后缀 Trie
  • AC 自动机：多模式匹配，基于 Trie + KMP

💡 面试高频问题

Q1: 为什么用 Trie 而不是哈希表？

答：前缀匹配是 Trie 的核心优势。哈希表只能做精确匹配，前缀匹配需要遍历所有键，时间复杂度 O(n×m)。而 Trie 是 O(m)。

Q2: Trie 的空间复杂度如何优化？

答：(1) 压缩 Trie，合并单分支链；(2) 用数组替代哈希表（已知字符集大小）；(3) 共享公共前缀天然节省空间。

Q3: 如何扩展支持删除操作？

答：增加引用计数或使用惰性删除。删除时检查节点是否还有其他子节点或是否为其他单词的结尾，决定是否真正删除节点。

Q4: 实际应用场景？

  • 搜索引擎自动补全
  • 输入法联想
  • 拼写检查/纠错
  • IP 路由最长前缀匹配
  • 基因组序列分析
