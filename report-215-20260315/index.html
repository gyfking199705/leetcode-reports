🎯 LeetCode 精讲 #215 - 数组中的第K个最大元素

📋 题目描述

原题链接: https://leetcode.cn/problems/kth-largest-element-in-an-array/

难度: 中等
标签: 数组、分治、快速选择、排序、堆（优先队列）

题目原文

给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例

示例 1：

输入: [3,2,1,5,6,4], k = 2
输出: 5
解释: 排序后数组为 [1,2,3,4,5,6]，第2大的元素是5

示例 2：

输入: [3,2,3,1,2,4,5,5,6], k = 4
输出: 4
解释: 排序后数组为 [1,2,2,3,3,4,5,5,6]，第4大的元素是4

约束条件

  • 1 <= k <= nums.length <= 10^5
  • -10^4 <= nums[i] <= 10^4

💡 解题思路

方法一：排序（最直观）

思路描述： 直接将数组排序，然后取倒数第k个元素即可。

为什么可行： 排序后数组是有序的，第k大的元素就是 nums[n-k]（升序排序）。

代码实现：

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        nums.sort()  # 升序排序
        return nums[len(nums) - k]  # 倒数第k个

复杂度分析：

  • 时间复杂度：O(n log n) —— 基于比较的排序下限
  • 空间复杂度：O(1) 或 O(log n) —— 取决于排序算法实现

缺点： 不满足题目要求的 O(n) 时间复杂度。

方法二：最小堆（优先队列）

思路描述： 维护一个大小为 k 的最小堆，遍历数组：

  1. 堆未满时，直接加入元素
  2. 堆满时，若当前元素 > 堆顶，弹出堆顶并加入当前元素

最终堆顶即为第k大的元素。

为什么可行： 最小堆的堆顶是堆中最小的元素。维护大小为k的堆，相当于保存了当前看到的最大的k个元素，堆顶就是这k个中最小的，即第k大。

代码实现：

import heapq

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        # 维护大小为k的最小堆
        min_heap = []
        
        for num in nums:
            if len(min_heap) < k:
                heapq.heappush(min_heap, num)
            elif num > min_heap[0]:
                # 当前元素比堆顶大，替换堆顶
                heapq.heapreplace(min_heap, num)
        
        return min_heap[0]

复杂度分析：

  • 时间复杂度：O(n log k) —— 每个元素最多一次堆操作
  • 空间复杂度：O(k) —— 堆的大小

优点： 适合数据流场景，不需要一次性加载所有数据。

方法三：快速选择（Quick Select）⭐ 推荐

核心思想： 借鉴快速排序的分区思想，但只递归处理包含目标元素的那一半。

算法步骤：

  1. 随机选择一个 pivot（枢轴）
  2. 将数组分区：小于pivot的放左边，大于pivot的放右边
  3. 比较 pivot 的位置与目标位置：
    • 如果 pivot 正好在第k大位置，返回 pivot
    • 如果 pivot 位置 > 目标位置，在左半部分继续查找
    • 如果 pivot 位置 < 目标位置，在右半部分继续查找

关键理解：

  • 第k大 = 升序排序后的第 n-k 个位置（0-based）
  • 分区后，pivot 的位置就是它在最终有序数组中的位置

代码实现：

import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        """
        快速选择算法 - 平均O(n)，最坏O(n^2)
        """
        n = len(nums)
        # 第k大 = 升序第n-k个位置
        return self.quickSelect(nums, 0, n - 1, n - k)
    
    def quickSelect(self, nums: List[int], left: int, right: int, target_idx: int) -> int:
        """
        在nums[left:right+1]中找第target_idx小的元素
        """
        if left == right:
            return nums[left]
        
        # 随机选择pivot，避免最坏情况
        pivot_idx = random.randint(left, right)
        
        # 分区，返回pivot最终位置
        final_pivot_idx = self.partition(nums, left, right, pivot_idx)
        
        if final_pivot_idx == target_idx:
            return nums[final_pivot_idx]
        elif final_pivot_idx < target_idx:
            # 目标在右半部分
            return self.quickSelect(nums, final_pivot_idx + 1, right, target_idx)
        else:
            # 目标在左半部分
            return self.quickSelect(nums, left, final_pivot_idx - 1, target_idx)
    
    def partition(self, nums: List[int], left: int, right: int, pivot_idx: int) -> int:
        """
        分区函数：将pivot放到正确位置，返回其最终索引
        所有小于pivot的元素放左边，大于的放右边
        """
        pivot_value = nums[pivot_idx]
        
        # 将pivot移到最右边
        nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
        
        # store_idx 指向小于pivot区域的下一个位置
        store_idx = left
        
        for i in range(left, right):
            if nums[i] < pivot_value:
                nums[store_idx], nums[i] = nums[i], nums[store_idx]
                store_idx += 1
        
        # 将pivot放到正确位置
        nums[right], nums[store_idx] = nums[store_idx], nums[right]
        
        return store_idx

迭代版本（避免递归栈溢出）：

import random
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        n = len(nums)
        target_idx = n - k  # 升序中的目标位置
        
        left, right = 0, n - 1
        
        while left <= right:
            pivot_idx = random.randint(left, right)
            final_pivot_idx = self.partition(nums, left, right, pivot_idx)
            
            if final_pivot_idx == target_idx:
                return nums[final_pivot_idx]
            elif final_pivot_idx < target_idx:
                left = final_pivot_idx + 1
            else:
                right = final_pivot_idx - 1
        
        return -1  # 不会执行到这里
    
    def partition(self, nums: List[int], left: int, right: int, pivot_idx: int) -> int:
        pivot_value = nums[pivot_idx]
        nums[pivot_idx], nums[right] = nums[right], nums[pivot_idx]
        
        store_idx = left
        for i in range(left, right):
            if nums[i] < pivot_value:
                nums[store_idx], nums[i] = nums[i], nums[store_idx]
                store_idx += 1
        
        nums[right], nums[store_idx] = nums[store_idx], nums[right]
        return store_idx

⏱️ 复杂度分析

方法 时间复杂度 空间复杂度 稳定性
排序 O(n log n) O(1) / O(log n) 稳定
最小堆 O(n log k) O(k) 稳定
快速选择 O(n) 平均 / O(n²) 最坏 O(1) 不稳定

快速选择时间复杂度证明：

  • 每次分区后，只需要处理一半的数据
  • T(n) = n + n/2 + n/4 + ... ≈ 2n = O(n)
  • 最坏情况（每次pivot都是最值）：O(n²)
  • 随机化pivot可极大降低最坏情况概率

🔑 关键要点

1. "第k大" vs "第k小" 的转换

第k大 = 升序第(n-k)个
第k小 = 升序第(k-1)个  # 0-based

2. 分区函数的核心逻辑

1. 选pivot → 2. 放末尾 → 3. 遍历交换 → 4. pivot归位

3. 快速选择 vs 快速排序

特性 快速排序 快速选择
目的 完全有序 找到第k个元素
递归 两边都递归 只递归一边
复杂度 O(n log n) O(n) 平均

4. 面试必考点

  • 为什么平均是O(n)？ 几何级数求和
  • 如何处理重复元素？ 上述代码天然支持
  • 最坏情况如何避免？ 随机化pivot / 三数取中

🎨 算法可视化

数组: [3, 2, 1, 5, 6, 4], k=2 (找第2大 = 第4小)

第1轮分区 (pivot=5):
[3, 2, 1, 4, 5, 6]  →  pivot索引=4, 目标索引=4 ✓
                              ↑
                          找到答案: 5

更复杂的情况: [7, 6, 5, 4, 3, 2, 1], k=3 (找第3大 = 第5小)

第1轮 (pivot=4):
[3, 2, 1, 4, 7, 6, 5]  pivot索引=3, 目标索引=5 → 去右边

第2轮 (pivot=6, 在[7,6,5]中):
[5, 6, 7]  →  [5, 6, 7]  pivot索引=1, 加上偏移量3+1=4 < 5 → 去右边

第3轮 (pivot=7):
[7]  →  找到！返回7

等等... 这里需要重新计算，实际第3大是5

修正：第3大 = 第5小 (0-based索引4)
[7, 6, 5, 4, 3, 2, 1] 排序后 [1, 2, 3, 4, 5, 6, 7]
                        索引:  0  1  2  3  4  5  6
                        第5小在索引4，值是5 ✓

💡 实战技巧

  1. Python内置函数：heapq.nsmallest(k, nums)[-1] 一行解决，但面试时不推荐
  2. 库函数调用：sorted(nums, reverse=True)[k-1] 同样简洁但不满足O(n)要求
  3. 边界测试：
    • k=1（最大值）
    • k=n（最小值）
    • 数组全部相同
    • 数组只有两个元素
