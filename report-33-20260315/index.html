🎯 LeetCode 精讲 #33 - 搜索旋转排序数组

📋 题目描述

来源: LeetCode 第33题
难度: Medium
链接: https://leetcode.cn/problems/search-in-rotated-sorted-array/

题目原文

整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1。

示例

示例 1:

输入: nums = [4,5,6,7,0,1,2], target = 0
输出: 4

示例 2:

输入: nums = [4,5,6,7,0,1,2], target = 3
输出: -1

示例 3:

输入: nums = [1], target = 0
输出: -1

约束条件

  • 1 <= nums.length <= 5000
  • -10^4 <= nums[i] <= 10^4
  • nums 中的每个值都 独一无二
  • 题目数据保证 nums 在预先未知的某个下标上进行了旋转
  • -10^4 <= target <= 10^4

💡 解题思路

一、核心洞察

这道题的关键在于：旋转后的数组，虽然整体不再有序，但总能找到一半是有序的。

举个例子：

原数组: [0, 1, 2, 4, 5, 6, 7]
旋转后: [4, 5, 6, 7, 0, 1, 2]
                ↑
              旋转点

观察旋转后的数组：

  • 以中间元素 7 为界，左边 [4,5,6,7] 是有序的
  • 右边 [0,1,2] 也是有序的（只是值整体较小）

关键性质：对于旋转排序数组，取中间元素后，左右两半必定有一个是有序的！

二、二分查找的变体

既然总有一半是有序的，我们可以利用这一点来判断 target 是否在这有序的一半中：

  1. 如果 target 在有序的那一半范围内 → 继续在这一半搜索
  2. 如果不在 → 在另一半搜索

三、图解算法

数组: [4, 5, 6, 7, 0, 1, 2], target = 0
       0  1  2  3  4  5  6

第1轮: left=0, right=6, mid=3
       nums[mid] = 7
       [4, 5, 6, 7] 有序？ nums[0] < nums[3] ✓ 是！
       target=0 在 [4,7] 范围内？ 否！
       → 在右半部分继续搜索
       
第2轮: left=4, right=6, mid=5
       nums[mid] = 1
       [0, 1, 2] 有序？ nums[4] < nums[6] ✓ 是！
       target=0 在 [0,2] 范围内？ 是！
       → 在左半部分继续搜索
       
第3轮: left=4, right=5, mid=4
       nums[mid] = 0
       找到！返回 4

📝 完整代码

Python 实现

from typing import List

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        """
        在旋转排序数组中搜索目标值
        时间复杂度: O(log n)
        空间复杂度: O(1)
        """
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = (left + right) // 2
            
            # 找到目标
            if nums[mid] == target:
                return mid
            
            # 判断哪一半是有序的
            if nums[left] <= nums[mid]:
                # 左半部分 [left, mid] 有序
                if nums[left] <= target < nums[mid]:
                    # target 在左半部分
                    right = mid - 1
                else:
                    # target 在右半部分
                    left = mid + 1
            else:
                # 右半部分 [mid, right] 有序
                if nums[mid] < target <= nums[right]:
                    # target 在右半部分
                    left = mid + 1
                else:
                    # target 在左半部分
                    right = mid - 1
        
        # 未找到
        return -1

🔍 关键细节分析

1. 有序性的判断

if nums[left] <= nums[mid]:
    # 左半有序

为什么用 <=？

  • 当 left == mid 时（只剩两个元素），左半部分仍然是"有序"的
  • 例如 [3, 1]，left=0, mid=0，此时左半部分只有一个元素，显然有序

2. 区间判断的细节

# 左半有序时
def is_in_left = nums[left] <= target < nums[mid]

# 右半有序时  
def is_in_right = nums[mid] < target <= nums[right]

注意开闭区间：

  • 左半有序时，nums[mid] 已经检查过不等于 target，所以用 <
  • 但 nums[left] 还没检查，所以用 <=（理论上可以都用 <，因为相等的情况已在前面处理）

3. 旋转点的处理

无论数组如何旋转，二分查找都能正确处理：

  • 旋转点在中间：标准情况
  • 旋转点在开头（未旋转）：退化为普通二分
  • 旋转点在结尾（旋转n-1次）：如 [2,3,4,5,6,7,0,1]

💡 面试高频考点

  1. 为什么时间复杂度还是 O(log n)？

    • 虽然数组被旋转了，但每次仍然能排除一半的元素
    • 有序的那一半可以直接判断 target 是否在其中

  2. 如何处理重复元素？（LeetCode 81）

    • 当 nums[left] == nums[mid] 时，无法判断哪边有序
    • 解决：left++，缩小范围，最坏退化为 O(n)

  3. 如何找到旋转点（最小元素）？

    • 与找 target 类似，判断哪边有序
    • 最小值一定在无序的那一边（除非全部有序）

  4. 如果要求用递归实现？

    • 思路相同，将迭代改为递归调用
