LeetCode 105 · 从前序与中序遍历序列构造二叉树

Medium 题号 105 • 递归 / 分治 / 哈希表 • 2026-04-29 00:06

🎯 题目描述

给定一棵树的前序遍历 preorder 与中序遍历 inorder，请构造这棵二叉树并返回其根节点。

核心前提：假设树中没有重复元素。（若有重复元素，则无法唯一确定结构）

💡 核心思路（点击展开）

前序遍历的顺序是 根 → 左 → 右，中序遍历的顺序是 左 → 根 → 右。

1定位根节点：前序数组的第一个元素就是整棵树的根。
2分割中序：在中序数组中找到根的位置，左边全部是左子树，右边全部是右子树。
3递归构造：根据左子树的节点数量，将前序数组也切成左右两部分，递归构建左右子树。
哈希表加速：在中序数组中查找根节点位置若用线性扫描，每次O(n)，总O(n²)。预先用哈希表存储值→索引的映射，查找降至O(1)，总时间O(n)。

🐍 Python 代码实现（点击展开）

最优解：哈希表 + 递归（O(n) 时间）

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        # 建立中序值 → 索引的映射，O(1) 查找
        index_map = {val: idx for idx, val in enumerate(inorder)}
        
        # pre_idx 用 nonlocal 追踪前序中的当前根位置
        self.pre_idx = 0
        
        def helper(in_left, in_right):
            # 中序区间无效，说明无节点
            if in_left > in_right:
                return None
            
            # 前序当前元素作为根
            root_val = preorder[self.pre_idx]
            root = TreeNode(root_val)
            self.pre_idx += 1
            
            # 在中序中定位根，分割左右子树
            in_root_idx = index_map[root_val]
            
            # 必须先构建左子树！前序是根→左→右
            root.left = helper(in_left, in_root_idx - 1)
            root.right = helper(in_root_idx + 1, in_right)
            
            return root
        
        return helper(0, len(inorder) - 1)

切片写法（简洁但不推荐，空间 O(n²)）

class Solution:
    def buildTree(self, preorder: List[int], inorder: List[int]) -> Optional[TreeNode]:
        if not preorder:
            return None
        
        root_val = preorder[0]
        root = TreeNode(root_val)
        idx = inorder.index(root_val)  # O(n) 查找
        
        # 切片产生新数组，浪费空间
        root.left = self.buildTree(preorder[1:1+idx], inorder[:idx])
        root.right = self.buildTree(preorder[1+idx:], inorder[idx+1:])
        
        return root
⚠️ 注意顺序：递归中必须先构建 left 再构建 right，因为前序遍历是「根→左→右」，pre_idx 指针必须按这个顺序消费元素。

📊 复杂度分析（点击展开）

指标哈希表优化版切片版
时间复杂度O(n)O(n²) — 每次 index() + 切片
空间复杂度O(n) — 哈希表 + 递归栈O(n²) — 大量切片新数组
递归深度O(h)，h 为树高度同左
最坏情况链表形态时 O(n) 栈深同左，且空间爆炸
面试建议：先写切片版的思路，展示理解；然后主动提出用索引范围 + 哈希表优化到 O(n)，这是加分项。

🎤 面试连环追问（点击展开）

Q1如果有重复元素怎么办？
无法唯一确定。需要额外信息，如前序+后序无法区分左右子树只有单节点的情况；或者给定 null 标记的序列化。
Q2只用前序+后序能构造吗？
不能唯一确定。当某个节点只有一棵子树时，无法判断是左还是右。例如 pre=[1,2], post=[2,1] 可以是 1→left→2 也可以是 1→right→2。
Q3如何迭代实现？
可以用栈模拟递归。维护当前处理的根节点，根据中序遍历的指针判断下一个前序元素是左孩子还是右兄弟。思路较复杂，面试中提及即可。
Q4空间能优化到 O(h) 吗？
如果不用哈希表，每次在中序中线性查找根节点，时间退化为 O(n²)，空间为 O(h)。trade-off 问题，通常不这么做。

🧠 记忆口诀（点击展开）

「前序找根，中序分左右，先左后右，递归到底」

📌 一句话总结

这道题是递归分治在二叉树上的经典应用。抓住「前序定根、中序分左右」的本质，配合哈希表将中序查找优化到 O(1)，即可在 O(n) 时间内重建整棵树。关键是递归时必须先左后右，因为前序遍历的顺序约束了子树的消费次序。
