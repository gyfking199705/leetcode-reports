📋 题目描述

给定两个字符串 s 和 t，返回 s 中涵盖 t 所有字符的最小子串。

如果不存在符合条件的子串，返回空字符串 ""。

注意：

  • 对于 t 中重复字符，我们寻找的子字符串中该字符数量必须不少于 t 中该字符数量
  • 如果 s 中存在这样的子串，我们保证它是唯一的答案

示例

示例 1：

输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小子串 "BANC" 包含了 'A'、'B'、'C' 各一个

示例 2：

输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 就是最小子串

示例 3：

输入：s = "a", t = "aa"
输出：""
解释：t 中两个 'a' 均被涵盖，但 s 只有一个 'a'，无法满足条件

💡 解题思路

核心思想：滑动窗口

这道题是滑动窗口算法的经典代表。滑动窗口的本质是维护一个动态的子数组/子串区间，通过调整左右边界来寻找满足条件的最优解。

为什么用滑动窗口？

  1. 连续性要求：我们要找的是连续的子串
  2. 单调性：当窗口扩大时，包含的字符只会增加不会减少；当满足条件后，我们可以尝试收缩左边界
  3. 最优子结构：一旦找到满足条件的窗口，其内部的子窗口如果也满足条件，则更优

算法流程

初始化：
  - need[char]: 记录t中每个字符的需求量
  - window[char]: 记录当前窗口中各字符的实际数量
  - valid: 记录窗口中满足需求的字符种类数

1. 右指针扩展窗口
   └─> 将s[right]加入window
   └─> 如果s[right]的需求量被满足，valid++

2. 当valid == need.size()，说明窗口已满足条件
   └─> 尝试收缩左边界：移动left，同时更新最小窗口
   └─> 如果移出的字符导致不满足条件，valid--

3. 重复步骤1-2，直到右指针遍历完s

Python 版本

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        from collections import defaultdict
        
        # 记录t中每个字符的需求量
        need = defaultdict(int)
        for c in t:
            need[c] += 1
        
        # 记录窗口中各字符的实际数量
        window = defaultdict(int)
        
        left, right = 0, 0          # 窗口边界
        valid = 0                   # 满足需求的字符种类数
        start, length = 0, float('inf')  # 最优解记录
        
        while right < len(s):
            # ===== 扩大窗口 =====
            c = s[right]
            right += 1
            
            # 更新窗口数据
            if c in need:
                window[c] += 1
                # 注意：只有当数量刚好满足时才增加valid
                if window[c] == need[c]:
                    valid += 1
            
            # ===== 收缩窗口 =====
            # 当所有字符都满足需求时，尝试缩小窗口
            while valid == len(need):
                # 更新最优解
                if right - left < length:
                    start = left
                    length = right - left
                
                # 移出左边界字符
                d = s[left]
                left += 1
                
                if d in need:
                    # 注意：只有当数量刚好不满足时才减少valid
                    if window[d] == need[d]:
                        valid -= 1
                    window[d] -= 1
        
        return "" if length == float('inf') else s[start:start + length]

Java 版本

class Solution {
    public String minWindow(String s, String t) {
        Map<Character, Integer> need = new HashMap<>();
        Map<Character, Integer> window = new HashMap<>();
        
        // 统计t中各字符需求量
        for (char c : t.toCharArray()) {
            need.put(c, need.getOrDefault(c, 0) + 1);
        }
        
        int left = 0, right = 0;
        int valid = 0;
        int start = 0, len = Integer.MAX_VALUE;
        
        while (right < s.length()) {
            // 扩大窗口
            char c = s.charAt(right);
            right++;
            
            if (need.containsKey(c)) {
                window.put(c, window.getOrDefault(c, 0) + 1);
                if (window.get(c).equals(need.get(c))) {
                    valid++;
                }
            }
            
            // 收缩窗口
            while (valid == need.size()) {
                // 更新最优解
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s.charAt(left);
                left++;
                
                if (need.containsKey(d)) {
                    if (window.get(d).equals(need.get(d))) {
                        valid--;
                    }
                    window.put(d, window.get(d) - 1);
                }
            }
        }
        
        return len == Integer.MAX_VALUE ? "" : s.substring(start, start + len);
    }
}

C++ 版本

class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char, int> need, window;
        
        for (char c : t) need[c]++;
        
        int left = 0, right = 0;
        int valid = 0;
        int start = 0, len = INT_MAX;
        
        while (right < s.size()) {
            char c = s[right];
            right++;
            
            if (need.count(c)) {
                window[c]++;
                if (window[c] == need[c]) valid++;
            }
            
            while (valid == need.size()) {
                if (right - left < len) {
                    start = left;
                    len = right - left;
                }
                
                char d = s[left];
                left++;
                
                if (need.count(d)) {
                    if (window[d] == need[d]) valid--;
                    window[d]--;
                }
            }
        }
        
        return len == INT_MAX ? "" : s.substr(start, len);
    }
};

🔍 关键点解析

1. valid 变量的巧妙设计

# 为什么用 valid 计数而不是直接比较两个字典？
if window[c] == need[c]:  # 刚好满足时
    valid += 1

原因：直接比较两个字典的时间复杂度是 O(字符集大小)，而维护 valid 计数只需要 O(1)。

2. 注意判断条件的时机

# 扩大时：刚满足条件时valid++
if window[c] == need[c]:
    valid += 1

# 缩小时：刚不满足条件时valid--
if window[d] == need[d]:
    valid -= 1

常见错误：写成 >= 或 <= 会导致 valid 计数错误。

3. 字符集处理

本题默认是 ASCII 字符，如果需要处理 Unicode：

  • Python：直接使用 dict 即可
  • Java/C++：继续使用 HashMap

⏱️ 复杂度分析

维度 复杂度 说明
时间 O(|S| + |T|) 左右指针各遍历s一次
空间 O(|字符集|) 两个哈希表，最多存储字符集大小个键

时间复杂度详解：

  • 右指针从0移动到 len(s)，每个元素访问一次：O(|S|)
  • 左指针总共也从0移动到 len(s)：O(|S|)
  • 初始化 need 哈希表：O(|T|)
  • 总复杂度：O(2|S| + |T|) = O(|S| + |T|)

🧠 面试常问

Q1: 如何证明滑动窗口的正确性？

使用双指针的单调性：右指针只会前进，左指针也只会前进。当找到一个满足条件的窗口时，所有以该窗口右边界结尾的更大窗口都不可能更优。

Q2: 如果t中有重复字符怎么办？

need哈希表会记录每个字符的需求量，window也会记录实际数量，只有当window[c] >= need[c]时才认为该字符满足条件。

Q3: 空间复杂度能优化吗？

如果字符集固定为大小写字母（52个），可以用固定大小的数组替代哈希表，空间复杂度降为O(1)。
