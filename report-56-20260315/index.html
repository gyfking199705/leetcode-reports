🎯 LeetCode 精讲 #56 - 合并区间

📋 题目描述

原题链接: https://leetcode.cn/problems/merge-intervals/

难度: 中等
标签: 数组、排序

题目原文

以数组 intervals 表示若干个区间的集合，其中单个区间为 intervals[i] = [starti, endi]。

请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

示例

示例 1：

输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠，将它们合并为 [1,6]。

示例 2：

输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 可被视为重叠区间（注意是闭区间）。

示例 3：

输入：intervals = [[1,4],[2,3]]
输出：[[1,4]]
解释：[2,3] 完全包含在 [1,4] 内，合并后仍是 [1,4]。

约束条件

  • 1 <= intervals.length <= 10^4
  • intervals[i].length == 2
  • 0 <= starti <= endi <= 10^4

💡 解题思路

🔍 核心洞察：先排序，再合并

这道题的关键在于：如何判断两个区间是否重叠？

两个区间 [a, b] 和 [c, d] 重叠的条件是什么？

重叠条件：b >= c  (第一个区间的右端点 >= 第二个区间的左端点)
         且 a <= d (第一个区间的左端点 <= 第二个区间的右端点)

但实际处理时，如果我们按照区间的左端点排序，问题会简化很多！

排序后，对于任意相邻的两个区间 intervals[i] 和 intervals[i+1]：

  • intervals[i][0] <= intervals[i+1][0] (左端点已排序)
  • 只需要判断 intervals[i][1] >= intervals[i+1][0] 是否成立

🎯 合并策略（排序后）

  1. 初始化：将第一个区间加入结果列表
  2. 遍历：对于每个新区间，与结果列表中最后一个区间比较
  3. 判断重叠：
    • 如果 新区间.start <= 当前区间.end → 有重叠，合并
      • 新区间 = [当前区间.start, max(当前区间.end, 新区间.end)]
    • 否则 → 无重叠，直接加入结果

📊 图示理解

原始区间：     [1,3]  [2,6]  [8,10]  [15,18]
                 ↓      ↓      ↓        ↓
按左端点排序后：[1,3]  [2,6]  [8,10]  [15,18]  (已排序)
                │      │      │        │
                └──────┘      │        │
                重叠合并      不重叠    不重叠
                   ↓          ↓        ↓
合并结果：      [1,6]       [8,10]  [15,18]

重叠判定详解：

区间A: [1, 3]    区间B: [2, 6]
       ────           ───────
         ↑←──重叠──→↑
       
因为 3 >= 2 (A的右端 >= B的左端)，所以重叠
合并后: [1, max(3,6)] = [1,6]

📝 参考代码

Python 实现

from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        """
        合并重叠区间
        
        核心思路：
        1. 按区间左端点排序
        2. 遍历排序后的区间，逐个判断是否与前一个区间重叠
        3. 重叠则合并，不重叠则加入结果
        """
        # 边界情况：空数组或只有一个区间
        if not intervals or len(intervals) <= 1:
            return intervals
        
        # Step 1: 按区间的左端点升序排序
        # key=lambda x: x[0] 表示按每个区间的第一个元素(左端点)排序
        intervals.sort(key=lambda x: x[0])
        
        # Step 2: 初始化结果列表，先将第一个区间加入
        merged = [intervals[0]]
        
        # Step 3: 遍历剩余的区间
        for i in range(1, len(intervals)):
            current = intervals[i]      # 当前要处理的区间
            last = merged[-1]           # 结果列表中最后一个区间
            
            # 判断当前区间与最后一个区间是否重叠
            # 重叠条件：当前区间的左端点 <= 最后一个区间的右端点
            if current[0] <= last[1]:
                # 有重叠，需要合并
                # 新区间的右端点 = max(当前右端点, 最后一个区间的右端点)
                # 左端点保持为 last[0]（因为已排序，last[0] <= current[0]）
                last[1] = max(last[1], current[1])
            else:
                # 无重叠，直接将当前区间加入结果
                merged.append(current)
        
        return merged

⏱️ 复杂度分析

复杂度 值 说明
时间复杂度 O(n log n) 排序 dominates，排序需要 O(n log n)，遍历需要 O(n)
空间复杂度 O(log n) ~ O(n) 排序栈空间 O(log n) 或 O(n)；结果数组 O(n)

详细解释：

  • 排序阶段：Python 的 Timsort 时间复杂度为 O(n log n)，是主要开销
  • 合并阶段：线性遍历 O(n)，每个区间最多被访问两次
  • 空间：如果不算输出数组，原地排序只需要 O(log n) 栈空间

🔑 关键要点

  1. 排序是核心：按左端点排序后，问题转化为简单的线性扫描
  2. 重叠判断：排序后只需比较 current.start <= last.end
  3. 合并操作：last.end = max(last.end, current.end) 处理完全包含的情况
  4. 边界处理：注意闭区间 [1,4] 和 [4,5] 也算重叠

🎯 面试高频考点

  1. 为什么先排序？ 不排序会怎样？

    • 不排序需要 O(n²) 两两比较，效率低
    • 排序后转化为线性问题，O(n log n) 可解

  2. 如何判断两个区间重叠？

    • 数学条件：max(a1, b1) <= min(a2, b2) 时两区间相交
    • 排序后简化：a2 <= b1（第二个区间左端点 <= 第一个区间右端点）

  3. 区间合并问题的扩展：

    • 最多同时有几个区间重叠？（会议室II）
    • 删除最少区间使剩下的不重叠？（无重叠区间）

  4. 实际应用场景：

    • 日程合并：合并日历中连续的事件
    • 资源调度：合并连续的资源使用时段
    • 数据压缩：合并相邻的数据块

💡 代码技巧

Pythonic 写法：

# 更简洁的实现（使用迭代器）
def merge(self, intervals):
    intervals.sort(key=lambda x: x[0])
    merged = []
    for interval in intervals:
        if not merged or merged[-1][1] < interval[0]:
            merged.append(interval)  # 不重叠，直接加入
        else:
            merged[-1][1] = max(merged[-1][1], interval[1])  # 合并
    return merged
