🎯 LeetCode 精讲 #3 - 无重复字符的最长子串

📋 题目描述

给定一个字符串 s，请你找出其中不含有重复字符的最长子串的长度。

示例 1：

输入: s = "abcabcbb"
输出: 3
解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3

示例 2：

输入: s = "bbbbb"
输出: 1
解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1

示例 3：

输入: s = "pwwkew"
输出: 3
解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3
     请注意，你的答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串

提示：

  • 0 <= s.length <= 5 * 10^4
  • s 由英文字母、数字、符号和空格组成

💡 解题思路

方法一：暴力解法（不推荐）

  • 核心思想：枚举所有子串，检查是否有重复字符，记录最长长度
  • 时间复杂度：O(n³) —— 枚举子串 O(n²)，检查重复 O(n)
  • 空间复杂度：O(min(m, n)) —— 需要集合存储字符

这种解法在面试中直接说出复杂度即可，不建议写代码实现。

方法二：滑动窗口 ⭐ 面试标准解法

  • 核心思想：维护一个"窗口"，窗口内的字符都是不重复的，通过移动左右边界来寻找最长窗口
详细解释：

什么是滑动窗口？

想象你有一个橡皮筋框，框住字符串中的一段。这个框可以左右伸缩：

  • 右边界右移：尝试纳入新字符
  • 左边界右移：当发现重复字符时，收缩窗口直到无重复

为什么用哈希表？

我们需要快速判断"某个字符是否在当前窗口中"，哈希表（Python中的set或dict）提供 O(1) 的查询效率。

算法步骤：

  1. 初始化：

    • left = 0：窗口左边界
    • max_len = 0：记录最大长度
    • char_set = set()：存储当前窗口中的字符

  2. 右指针遍历字符串（right 从 0 到 len(s)-1）：

    • 情况A：s[right] 不在 char_set 中

      • 说明可以安全地加入窗口
      • char_set.add(s[right])
      • 更新最大长度：max_len = max(max_len, right - left + 1)

    • 情况B：s[right] 已在 char_set 中

      • 说明出现了重复，需要收缩左边界
      • 关键操作：不断移动 left，并从 char_set 中移除 s[left]，直到 s[right] 不再在集合中
      • 然后将 s[right] 加入窗口

  3. 返回：max_len

图示过程（以 "abcabcbb" 为例）：

步骤1: left=0, right=0, 窗口="a", 集合={a}, max_len=1
步骤2: left=0, right=1, 窗口="ab", 集合={a,b}, max_len=2
步骤3: left=0, right=2, 窗口="abc", 集合={a,b,c}, max_len=3
步骤4: left=0, right=3, s[3]='a' 重复！
       移动left: 移除'a', left=1, 集合={b,c}
       现在'a'不在集合中，加入'a', 窗口="bca", max_len=3
步骤5: left=1, right=4, s[4]='b' 重复！
       移动left: 移除'b', left=2, 集合={c,a}
       加入'b', 窗口="cab", max_len=3
...以此类推

方法三：优化的滑动窗口（使用哈希表记录位置）⭐⭐ 进阶解法

上面的解法在遇到重复时，需要逐个移除字符。我们可以优化：直接用哈希表记录每个字符的最新位置，遇到重复时，直接将左边界跳到重复位置的下一个。

优化点：

  • 使用 char_index 字典记录字符 → 索引的映射
  • 当发现 s[right] 重复时，left 直接跳到 char_index[s[right]] + 1
  • 但注意：left 只能右移，不能左移（所以需要用 max(left, ...)）

📝 参考代码

标准滑动窗口（推荐面试写法）

def lengthOfLongestSubstring(s: str) -> int:
    """
    滑动窗口解法 - 使用集合判断重复
    时间复杂度: O(n) - 每个字符最多被访问两次（右指针一次，左指针一次）
    空间复杂度: O(min(m, n)) - m是字符集大小
    """
    char_set = set()      # 存储当前窗口中的字符
    left = 0              # 左边界
    max_len = 0           # 最大长度
    
    for right in range(len(s)):
        # 如果当前字符已在窗口中，收缩左边界直到无重复
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        
        # 将当前字符加入窗口
        char_set.add(s[right])
        
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len

优化的滑动窗口（进阶）

def lengthOfLongestSubstringOptimized(s: str) -> int:
    """
    优化版滑动窗口 - 使用字典记录字符位置
    时间复杂度: O(n)
    空间复杂度: O(min(m, n))
    
    优化点：遇到重复时，左边界直接跳到重复位置的下一个，无需逐个移除
    """
    char_index = {}       # 字符 -> 最新索引
    left = 0              # 左边界
    max_len = 0           # 最大长度
    
    for right in range(len(s)):
        # 如果当前字符已出现过，且其位置在当前窗口内
        if s[right] in char_index and char_index[s[right]] >= left:
            # 左边界直接跳到重复位置的下一个
            left = char_index[s[right]] + 1
        
        # 更新字符的最新位置
        char_index[s[right]] = right
        
        # 更新最大长度
        max_len = max(max_len, right - left + 1)
    
    return max_len

数组优化版本（针对ASCII字符集）

def lengthOfLongestSubstringArray(s: str) -> int:
    """
    针对ASCII字符集的极致优化 - 使用数组代替哈希表
    时间复杂度: O(n)
    空间复杂度: O(128) = O(1) - 固定大小的数组
    """
    # ASCII共128个字符，用-1表示"未出现过"
    char_index = [-1] * 128
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        char_code = ord(s[right])
        
        # 如果字符在当前窗口内出现过
        if char_index[char_code] >= left:
            left = char_index[char_code] + 1
        
        char_index[char_code] = right
        max_len = max(max_len, right - left + 1)
    
    return max_len

⏱️ 复杂度分析

解法 时间复杂度 空间复杂度 说明
暴力枚举 O(n³) O(min(m, n)) 面试提及即可，不写代码
标准滑动窗口 O(n) O(min(m, n)) ⭐ 推荐面试写法
优化滑动窗口 O(n) O(min(m, n)) 跳步优化，代码稍复杂
数组优化版 O(n) O(1) 针对ASCII的极致优化

时间复杂度 O(n) 的推导：

  • 右指针 right 遍历整个字符串一次：O(n)
  • 左指针 left 最多也遍历整个字符串一次：O(n)
  • 每个字符最多被访问两次，所以总体是 O(2n) = O(n)

🔑 关键要点

  1. 滑动窗口的核心思想

    • 维护一个"有效区间"，区间内满足特定条件（本题是"无重复字符"）
    • 右边界主动扩张，左边界被动收缩

  2. 为什么用 while 而不是 if 来收缩左边界？

    • 示例：s = "abba"
    • 当 right=3（第二个'a'），如果只移一次 left，窗口变成 "ba"，仍包含 'a'
    • 需要一直移直到 'a' 被移除

  3. 易错点：优化版的 max(left, ...)

    # 错误写法（可能让left左移！）
left = char_index[s[right]] + 1

# 正确写法（left只能右移）
left = max(left, char_index[s[right]] + 1)

    原因：char_index 记录的是"出现过"，但这个位置可能在当前窗口左边，不需要移动

  4. 子串 vs 子序列

    • 子串（Substring）：必须连续，如 "abc" 是 "abcde" 的子串
    • 子序列（Subsequence）：不需要连续，如 "ace" 是 "abcde" 的子序列
    • 本题要求的是子串，不是子序列

🎓 面试追问

Q1：如果字符串包含Unicode字符，代码需要怎么改？

  • 答：Python的字符串天然支持Unicode，无需修改。但如果用数组优化版，需要将数组大小扩展到Unicode范围（不太实际），还是用哈希表更合适。

Q2：如果不只是判断ASCII字符，而是包括中文、emoji等？

  • 答：使用标准滑动窗口的哈希表版本即可，Python的dict和set都能处理Unicode字符。

Q3：如果要返回最长子串本身，而不仅仅是长度？

  • 答：在更新max_len时，同时记录start_index = left，最后返回s[start_index:start_index + max_len]
