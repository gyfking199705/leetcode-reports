🎯 LeetCode 精讲 #141 - 环形链表

📋 题目描述

来源: LeetCode 第141题
难度: Easy
链接: https://leetcode.cn/problems/linked-list-cycle/

题目原文

给你一个链表的头节点 head ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：pos 不作为参数进行传递。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true ；否则，返回 false 。

示例

示例 1:

输入: head = [3,2,0,-4], pos = 1
输出: true
解释: 链表中有一个环，其尾部连接到第二个节点。

示例 2:

输入: head = [1,2], pos = 0
输出: true
解释: 链表中有一个环，其尾部连接到第一个节点。

示例 3:

输入: head = [1], pos = -1
输出: false
解释: 链表中没有环。

约束条件

  • 链表中节点的数目范围是 [0, 10⁴]
  • -10⁵ <= Node.val <= 10⁵
  • pos 为 -1 或者链表中的一个有效索引

💡 解题思路

方法一：哈希表法（直观但非最优）

核心思想: 遍历链表，用哈希表记录访问过的节点。如果再次遇到已访问的节点，说明有环。

# Definition for singly-linked list.
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        visited = set()
        current = head
        
        while current:
            if current in visited:
                return True  # 发现环！
            visited.add(current)
            current = current.next
        
        return False  # 遍历结束，无环

复杂度分析:

  • 时间复杂度: O(n) - 最坏情况遍历所有节点
  • 空间复杂度: O(n) - 需要存储访问过的节点

方法二：快慢指针法（Floyd判圈算法）⭐ 推荐

核心思想: 使用两个指针，慢指针每次走1步，快指针每次走2步。如果存在环，快指针最终会追上慢指针；如果无环，快指针会先到达末尾。

为什么快指针一定会追上慢指针？

想象一下操场跑步：

  • 慢指针速度: 1米/秒
  • 快指针速度: 2米/秒
  • 相对速度: 1米/秒（快指针以每秒1米的速度接近慢指针）

如果操场周长是C，快指针落后慢指针d米，那么经过d秒后，快指针就会追上慢指针。

# Definition for singly-linked list.
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        # 边界情况：空链表或只有一个节点
        if not head or not head.next:
            return False
        
        # 初始化快慢指针
        slow = head  # 慢指针，每次走1步
        fast = head.next  # 快指针，每次走2步
        
        # 当快指针能继续前进时循环
        while fast and fast.next:
            if slow == fast:
                return True  # 快慢指针相遇，有环！
            
            slow = slow.next       # 慢指针走1步
            fast = fast.next.next  # 快指针走2步
        
        return False  # 快指针到达末尾，无环

复杂度分析:

  • 时间复杂度: O(n) - 无环时快指针到达末尾；有环时，快指针最多走2n步就会追上慢指针
  • 空间复杂度: O(1) - 只使用了两个指针，常数额外空间

🔍 代码详解

快慢指针执行过程可视化

假设链表: 3 → 2 → 0 → -4 → (回到2，形成环)

初始状态:
slow → 3, fast → 2

第1轮:
slow: 3 → 2
fast: 2 → 0 → -4
状态: slow=2, fast=-4

第2轮:
slow: 2 → 0
fast: -4 → 2 → 0  (绕了一圈)
状态: slow=0, fast=0  ✨ 相遇！返回 True

另一种初始化方式（更常见）

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = fast = head  # 都从头部开始
        
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                return True
        
        return False

注意: 这种写法第一次迭代就会比较（head == head.next，一般不会相等），需要多走一轮才能正确判断。

🎓 进阶知识

1. 为什么快指针走2步，而不是3步、4步？

走2步是最优选择：

  • 2步: 保证一定能相遇，且代码简洁
  • 3步及以上: 仍能找到环，但可能跳过慢指针（需要更复杂的判断）

2. 如何找到环的入口？（LeetCode 142）

当快慢指针相遇后：

  1. 将慢指针移回头部
  2. 两个指针都以1步速度前进
  3. 再次相遇的点就是环的入口

数学证明: 设环入口距离头部a步，相遇点距离入口b步，环长为c。

  • 慢指针走的距离: a + b
  • 快指针走的距离: a + b + kc (k圈)
  • 因为快指针是慢指针的2倍: 2(a+b) = a + b + kc
  • 化简: a = kc - b = (k-1)c + (c-b)
  • 结论: 从头部走a步 = 从相遇点走c-b步（即回到入口）
# LeetCode 142 - 环形链表 II
def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
    slow = fast = head
    
    # 第一步：找相遇点
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            # 第二步：找入口
            slow = head  # 慢指针回头部
            while slow != fast:
                slow = slow.next
                fast = fast.next
            return slow  # 相遇点就是入口
    
    return None  # 无环

3. 如何计算环的长度？

找到相遇点后，让一个指针不动，另一个指针继续前进，再次回到该点所需的步数就是环长。

💬 面试高频考点

Q1: 快慢指针为什么一定能相遇？
A: 如果存在环，快指针相对慢指针以1步/轮的速度追赶，环的长度是有限的，所以一定能在某轮追上。

Q2: 相遇时慢指针一定没有走完一圈吗？
A: 是的。快指针速度是慢指针的2倍，当慢指针进入环后，快指针最多落后一圈，在慢指针走完第一圈之前就会被追上。

Q3: 为什么空间复杂度是O(1)？
A: 只用了slow和fast两个指针，不随链表长度增加而增加额外空间。

Q4: 如果要求找到环的长度，怎么做？
A: 相遇后，让一个指针不动，另一个指针继续走，统计再次回到该点的步数。

🎯 总结

LeetCode 141 是理解**快慢指针（Floyd判圈算法）**的经典入门题。

核心要点:

  1. 快慢指针是解决链表环问题的标准方法
  2. 快指针每次走2步，慢指针每次走1步
  3. 相遇则有环，快指针到达末尾则无环
  4. 时间O(n)，空间O(1)，非常高效
