🎯 LeetCode 精讲 #15 - 三数之和

📋 题目描述

难度: 中等
链接: https://leetcode.cn/problems/3sum/

给定一个整数数组 nums，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k，同时还满足 nums[i] + nums[j] + nums[k] == 0。

请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例 1：

输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
- nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0
- nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0（重复，舍去）
- nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0

示例 2：

输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0

示例 3：

输入：nums = [0,0,0]
输出：[[0,0,0]]

💡 解题思路

核心思想：排序 + 双指针

这道题是「两数之和」的升级版。如果直接暴力枚举三个数，时间复杂度是 O(n³)，显然不可接受。

关键观察：

  1. 排序：将数组排序后，我们可以利用有序性来剪枝和去重
  2. 固定一个数，转化为两数之和：固定 nums[i]，问题变成在 i 之后的区间找两数之和等于 -nums[i]
  3. 双指针：对于两数之和，用双指针可以在 O(n) 时间解决

详细步骤

1. 对数组进行排序
2. 遍历数组，固定第一个数 nums[i]
   - 如果 nums[i] > 0，由于数组有序，后面的数都 > 0，三数之和不可能为 0，直接 break
   - 如果 nums[i] == nums[i-1]，跳过（去重）
   
3. 在 i 之后的区间 [i+1, n-1] 使用双指针找两数之和等于 -nums[i]
   - left = i + 1, right = n - 1
   - 计算 sum = nums[i] + nums[left] + nums[right]
   - 如果 sum == 0：记录结果，并移动左右指针去重
   - 如果 sum < 0：left++（需要更大的数）
   - 如果 sum > 0：right--（需要更小的数）

去重策略（重点！）

这道题最容易出错的就是去重。我们分三个层次去重：

  1. 外层去重：nums[i] 不能和前一个相同

    if i > 0 and nums[i] == nums[i-1]:
    continue

  2. 左指针去重：找到一组解后，left 要跳过重复元素

    while left < right and nums[left] == nums[left+1]:
    left += 1

  3. 右指针去重：同理

    while left < right and nums[right] == nums[right-1]:
    right -= 1

📝 参考代码

from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        """
        三数之和 - 排序 + 双指针
        
        思路：
        1. 先排序，便于去重和使用双指针
        2. 固定第一个数，在剩余区间用双指针找两数之和为 -nums[i]
        3. 多层去重避免重复结果
        """
        n = len(nums)
        if n < 3:
            return []
        
        # 排序是双指针的前提
        nums.sort()
        result = []
        
        # 固定第一个数
        for i in range(n - 2):  # 留出两个位置给 left 和 right
            # 剪枝1：当前数已大于0，后面都是正数，不可能和为0
            if nums[i] > 0:
                break
            
            # 去重1：跳过重复的 nums[i]
            # 注意要和前一个比，不是和后一个比
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            
            # 双指针初始化
            left, right = i + 1, n - 1
            target = -nums[i]  # 两数之和的目标
            
            while left < right:
                two_sum = nums[left] + nums[right]
                
                if two_sum == target:
                    # 找到一组解
                    result.append([nums[i], nums[left], nums[right]])
                    
                    # 去重2：跳过重复的 left
                    while left < right and nums[left] == nums[left + 1]:
                        left += 1
                    
                    # 去重3：跳过重复的 right
                    while left < right and nums[right] == nums[right - 1]:
                        right -= 1
                    
                    # 移动到下一个不同的位置
                    left += 1
                    right -= 1
                    
                elif two_sum < target:
                    # 和太小，需要更大的数，左指针右移
                    left += 1
                else:
                    # 和太大，需要更小的数，右指针左移
                    right -= 1
        
        return result

⏱️ 复杂度分析

指标 复杂度 说明
时间复杂度 O(n²) 排序 O(n log n) + 双层遍历 O(n²)
空间复杂度 O(1) / O(log n) 结果数组不计入；排序可能使用 O(log n) 栈空间

详细推导：

  • 外层循环遍历 n 个数
  • 内层双指针最多遍历 n 个数
  • 总时间：O(n²)，其中排序的 O(n log n) 被主导

🔑 关键要点

  1. 先排序 - 这是使用双指针和去重的前提

  2. 去重时机 - 最容易错的地方：

    • nums[i] 去重：i > 0 and nums[i] == nums[i-1]
    • 找到解后才对 left/right 去重

  3. 剪枝优化 - if nums[i] > 0: break，因为数组有序

  4. 双指针移动 - 根据三数之和与0的比较决定移动方向

  5. 边界条件：

    • 数组长度小于3直接返回
    • 注意指针交叉的判断 left < right

🎓 面试追问

Q1: 为什么不能用哈希表像两数之和那样做？
A: 可以，但去重非常麻烦。三数之和要求返回的是不重复的三元组值，而不是下标，哈希表去重逻辑复杂且空间复杂度高。

Q2: 如果要求返回下标而不是数值，怎么做？
A: 那就不能用排序了，需要用哈希表存储值到下标的映射，但去重仍然很麻烦，可能需要使用集合去重。

Q3: 时间复杂度还能优化吗？
A: 理论下界是 O(n²)，因为最坏情况下确实可能有 O(n²) 个解（比如大量0的情况）。
