📋 题目描述

你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1 。

在选修某些课程之前需要一些先修课程。先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则 必须 先学习课程 bi 。

  • 例如，先修课程对 [0, 1] 表示：想要学习课程 0 ，你需要先完成课程 1 。

请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。

示例

示例 1:

输入: numCourses = 2, prerequisites = [[1,0]]
输出: true
解释: 总共有 2 门课程。学习课程 1 之前，你需要完成课程 0。这是可能的。

示例 2:

输入: numCourses = 2, prerequisites = [[1,0],[0,1]]
输出: false
解释: 想要学习课程 0，需要先完成课程 1；但学习课程 1 之前，又需要先完成课程 0。这是不可能的。

约束条件

  • 1 <= numCourses <= 2000
  • 0 <= prerequisites.length <= 5000
  • prerequisites[i].length == 2
  • 0 <= ai, bi < numCourses
  • prerequisites[i] 中的所有课程对 互不相同

💡 解题思路

核心问题识别

这道题的本质是判断有向图是否存在环。把课程看作图中的节点，先修关系看作有向边，那么：

  • 如果图中存在环 → 无法完成所有课程 → 返回 false
  • 如果图中无环 → 可以按拓扑顺序完成 → 返回 true

什么是拓扑排序？

拓扑排序是对有向无环图（DAG）的节点进行线性排序，使得对于图中的每条有向边 (u, v)，节点 u 在排序中始终位于节点 v 之前。

简单说：先修课排前面，后修课排后面。

两种经典解法

方法 核心思想 时间复杂度 空间复杂度
Kahn算法（BFS） 不断移除入度为0的节点 O(V + E) O(V + E)
DFS后序遍历 深度优先，检测回边 O(V + E) O(V + E)

其中 V = 课程数，E = 先修关系数

🔧 方法一：Kahn算法（BFS拓扑排序）

算法步骤

  1. 建图：构建邻接表，同时计算每个节点的入度（有多少先修课）
  2. 初始化：将所有入度为0的节点加入队列（这些课没有先修要求，可以直接学）
  3. BFS处理：
    • 弹出队列中的课程（表示已完成）
    • 将该课程的所有后继课程的入度减1
    • 如果某后继课程入度变为0，加入队列
  4. 判断结果：如果成功处理的课程数等于总课程数，说明无环，返回 true

为什么这样能检测环？

  • 有向无环图（DAG）必然存在至少一个入度为0的节点
  • 环上的所有节点入度都 ≥ 1，永远无法被处理
  • 所以如果能处理完所有节点，说明图中无环

代码实现

from collections import deque
from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 1. 建图 + 计算入度
        graph = [[] for _ in range(numCourses)]  # 邻接表
        in_degree = [0] * numCourses              # 入度数组
        
        for course, pre in prerequisites:
            graph[pre].append(course)   # pre -> course 的先修关系
            in_degree[course] += 1       # course 的入度+1
        
        # 2. 将所有入度为0的节点加入队列
        queue = deque()
        for i in range(numCourses):
            if in_degree[i] == 0:
                queue.append(i)
        
        # 3. BFS处理
        processed = 0  # 记录已处理的课程数
        
        while queue:
            curr = queue.popleft()
            processed += 1
            
            # 处理curr的所有后继课程
            for next_course in graph[curr]:
                in_degree[next_course] -= 1  # 入度减1
                if in_degree[next_course] == 0:
                    queue.append(next_course)
        
        # 4. 判断是否所有课程都被处理
        return processed == numCourses

# 建图后: graph[0]=[1], graph[1]=[2], graph[2]=[3]

🔧 方法二：DFS检测环

算法思想

使用三种状态标记节点：

  • 0（未访问）：白色，还未探索
  • 1（访问中）：灰色，当前在递归栈中
  • 2（已访问）：黑色，已完成探索

如果在DFS过程中遇到一个状态为1（灰色）的节点，说明遇到了回边，存在环。

代码实现

from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        # 建图
        graph = [[] for _ in range(numCourses)]
        for course, pre in prerequisites:
            graph[pre].append(course)
        
        # 0=未访问, 1=访问中(在递归栈), 2=已访问
        state = [0] * numCourses
        
        def has_cycle(node: int) -> bool:
            """以node为起点进行DFS，检测是否有环"""
            if state[node] == 1:  # 遇到灰色节点，存在环！
                return True
            if state[node] == 2:  # 遇到黑色节点，已处理过，无环
                return False
            
            # 标记为灰色（访问中）
            state[node] = 1
            
            # DFS所有后继节点
            for next_course in graph[node]:
                if has_cycle(next_course):
                    return True
            
            # 标记为黑色（已完成）
            state[node] = 2
            return False
        
        # 对每个未访问的节点进行DFS
        for i in range(numCourses):
            if state[i] == 0:  # 只处理白色节点
                if has_cycle(i):
                    return False
        
        return True

📊 复杂度分析

指标 Kahn(BFS) DFS
时间复杂度 O(V + E) O(V + E)
空间复杂度 O(V + E) O(V + E)

详细分析

时间复杂度 O(V + E)：

  • 建图需要遍历所有边：O(E)
  • BFS/DFS每个节点和边只访问一次：O(V + E)

空间复杂度 O(V + E)：

  • 邻接表存储图：O(V + E)
  • 入度数组/访问状态数组：O(V)
  • 队列/递归栈：最坏O(V)

🎯 关键要点总结

1. 建图的两种视角

# 方式1: prerequisites[i] = [ai, bi] 表示 bi -> ai (先修指向后续)
# 这是题目的本意，也是上面代码用的方式
graph[bi].append(ai)

# 方式2: 也可以反过来建图，但逻辑要对应调整

2. 为什么Kahn算法能找出所有拓扑排序？

  • 每次取入度为0的节点，保证了先修关系
  • 如果某时刻有多个入度为0的节点，任选一个都能得到合法的拓扑序
  • 所有可能的选法组合起来就是所有的拓扑排序

3. 两种方法的适用场景

  • Kahn（BFS）：天然适合找"最短完成时间"、可以并行处理的课程
  • DFS：代码更简洁，适合需要输出具体拓扑序的场景（逆后序就是拓扑序）

4. 面试常考变形

  • 输出任意一个合法的修课顺序 → 记录BFS/DFS顺序即可
  • 输出所有可能的修课方案 → 回溯
  • 最少需要几个学期修完（并行处理） → BFS分层

💬 面试官追问

Q: 如果要输出一个可行的选课顺序，怎么做？

A: BFS方法中用一个数组记录出队顺序，最后如果 len(order) == numCourses 就返回order；DFS方法用逆后序（节点变黑时加入结果），最后reverse结果就是拓扑序。

Q: 如果课程有权重（修课需要时间），如何计算最短完成时间？

A: 拓扑排序后动态规划，dp[i] = max(dp[j] + time[i])，其中j是i的所有先修课。

Q: 如果数据规模很大（10^5级别），有什么优化？

A: 本题复杂度已经是线性的O(V+E)，无法更优。但要注意Python的递归深度限制（DFS时），可以改迭代或设 sys.setrecursionlimit()。
