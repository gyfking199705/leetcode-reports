🎯 LeetCode 精讲 #199 - 二叉树的右视图

📋 题目描述

来源: LeetCode 第199题
难度: Medium
链接: https://leetcode.cn/problems/binary-tree-right-side-view/

题目原文

给定一个二叉树的 根节点 root，想象自己站在它的右侧，按照从顶部到底部的顺序，返回从右侧所能看到的节点值。

示例

示例 1:

输入: root = [1,2,3,null,5,null,4]
输出: [1,3,4]
解释:
       1            ← 第0层，看到 1
     /   \
    2     3         ← 第1层，看到 3
     \     \
      5     4       ← 第2层，看到 4

示例 2:

输入: root = [1,null,3]
输出: [1,3]
解释:
       1            ← 看到 1
        \
         3         ← 看到 3

示例 3:

输入: root = []
输出: []

约束条件

  • 二叉树的节点个数的范围是 [0, 100]
  • -100 <= Node.val <= 100

💡 解题思路

核心观察

"右视图"的本质是什么？

对于每一层（深度），我们能看到的是该层最右边的节点

        1              ← 第0层最右: 1
      /   \
     2     3           ← 第1层最右: 3
    / \   / \
   4   5 6   7         ← 第2层最右: 7

所以问题转化为：按层遍历二叉树，取每层最后一个节点。

方法一：BFS层序遍历（推荐）

思路: 使用标准的层序遍历（BFS），记录每一层的最后一个节点。

算法步骤:
1. 如果根节点为空，返回空列表
2. 初始化队列，将根节点入队
3. 当队列不为空时:
   a. 记录当前层的节点数量 n
   b. 遍历当前层的 n 个节点:
      - 出队一个节点
      - 如果是当前层最后一个节点(i == n-1)，加入结果
      - 将左右子节点入队(如果存在)
4. 返回结果列表

可视化过程:

初始: queue = [1]

第0层: n=1
      出队 1 → 是最后一个，res=[1]
      入队 2, 3
      queue = [2, 3]

第1层: n=2
      出队 2 → 不是最后一个
      入队 5
      出队 3 → 是最后一个，res=[1,3]
      入队 4
      queue = [5, 4]

第2层: n=2
      出队 5 → 不是最后一个
      出队 4 → 是最后一个，res=[1,3,4]
      queue = []

结束，返回 [1,3,4]

方法二：DFS深度优先搜索（先右后左）

思路: 采用DFS，但关键技巧是先访问右子节点，再访问左子节点。这样第一次访问某一层时，就是从右边看到的节点。

算法步骤:
1. 定义递归函数 dfs(node, depth):
   - 如果节点为空，返回
   - 如果 depth 等于结果列表长度，说明这是该层第一个访问的节点(从右看)
     → 将节点值加入结果
   - 先递归右子节点: dfs(node.right, depth+1)
   - 再递归左子节点: dfs(node.left, depth+1)

2. 调用 dfs(root, 0)
3. 返回结果

为什么先右后左？

        1
      /   \
     2     3
    /       \
   4         5

先访问右子树的顺序: 1 → 3 → 5 → 2 → 4
按深度记录首次访问: depth0:1, depth1:3, depth2:5
正好是右视图！

📝 参考代码

方法一：BFS层序遍历

from collections import deque
from typing import List, Optional

# 定义二叉树节点
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        """
        BFS层序遍历解法
        时间复杂度: O(n) - 每个节点访问一次
        空间复杂度: O(w) - w为树的最大宽度，队列最大长度
        """
        if not root:
            return []
        
        result = []
        queue = deque([root])  # 初始化队列
        
        while queue:
            # 当前层的节点数量
            level_size = len(queue)
            
            for i in range(level_size):
                node = queue.popleft()
                
                # 当前层的最后一个节点，就是右视图能看到的
                if i == level_size - 1:
                    result.append(node.val)
                
                # 将子节点加入队列
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
        
        return result

方法二：DFS深度优先（先右后左）

from typing import List, Optional

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        """
        DFS解法：先访问右子树，这样每层第一个访问的就是右视图
        时间复杂度: O(n) - 每个节点访问一次
        空间复杂度: O(h) - h为树高度，递归栈深度
        """
        result = []
        
        def dfs(node: Optional[TreeNode], depth: int):
            if not node:
                return
            
            # 关键：如果当前深度等于结果长度，说明是该层第一个访问的节点
            # 由于我们先访问右子树，第一个访问的就是右视图节点
            if depth == len(result):
                result.append(node.val)
            
            # 先访问右子树！这是关键
            dfs(node.right, depth + 1)
            # 再访问左子树
            dfs(node.left, depth + 1)
        
        dfs(root, 0)
        return result

方法三：DFS（记录每层最大值，通用模板）

from typing import List, Optional

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> List[int]:
        """
        DFS通用模板：使用字典记录每层的节点
        最后取每层的最后一个（最右）
        """
        from collections import defaultdict
        
        level_nodes = defaultdict(list)
        
        def dfs(node: Optional[TreeNode], depth: int):
            if not node:
                return
            
            level_nodes[depth].append(node.val)
            dfs(node.left, depth + 1)
            dfs(node.right, depth + 1)
        
        dfs(root, 0)
        
        # 按层取最后一个
        result = []
        for depth in sorted(level_nodes.keys()):
            result.append(level_nodes[depth][-1])
        
        return result

⏱️ 复杂度分析

方法 时间复杂度 空间复杂度 说明
BFS O(n) O(w) w是树的最大宽度，最坏情况完全二叉树O(n)
DFS(先右后左) O(n) O(h) h是树高度，最坏情况退化为链表O(n)
DFS(通用模板) O(n) O(n) 需要存储所有节点，空间开销大

BFS vs DFS选择:

  • BFS: 直观易懂，是层序遍历的自然延伸
  • DFS: 代码更简洁，空间复杂度更优（对于平衡树）

🔑 关键要点

1. BFS核心技巧

level_size = len(queue)  # 必须先记录！
for i in range(level_size):
    node = queue.popleft()
    if i == level_size - 1:  # 当前层最后一个
        result.append(node.val)

2. DFS核心技巧

# 先右后左，每层第一个就是答案
dfs(node.right, depth + 1)
dfs(node.left, depth + 1)

# 判断条件：深度等于结果长度时，是该层第一个
depth == len(result)

3. 常见错误

错误1: BFS中不记录level_size

# 错误！这样无法区分层
while queue:
    node = queue.popleft()  # 不知道是哪一层的

错误2: DFS先左后右

# 这样得到的是左视图！
dfs(node.left, depth + 1)
dfs(node.right, depth + 1)

4. 扩展：左视图怎么求？

# BFS: 取每层的第一个
if i == 0:
    result.append(node.val)

# DFS: 先左后右
dfs(node.left, depth + 1)
dfs(node.right, depth + 1)

🎯 面试高频考点

  1. BFS和DFS如何选择？

    • 按层处理的题目首选BFS
    • 但本题DFS代码更简洁，可以展示两种解法

  2. DFS先右后左的原理是什么？

    • 确保每层第一个访问的是最右节点
    • depth == len(result) 判断是第几次访问该层

  3. 空间复杂度对比？

    • BFS: O(w)，最宽层的节点数
    • DFS: O(h)，树的高度
    • 平衡树DFS更优，极端情况BFS可能更优

  4. 扩展问题：

    • 如果要求左视图？（先左后右 或 取每层第一个）
    • 如果要求俯视图？（最上面能看到的节点）
